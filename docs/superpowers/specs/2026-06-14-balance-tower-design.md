# Balance Tower — дизайн-документ

Дата: 2026-06-14
Статус: черновик, согласован устно, ждёт ревью спецификации.

## Краткое описание

Аркада на физике: игрок строит башню, роняя объекты сверху. Объект едет
влево-вправо, по тапу падает на башню «как есть» — ничего не обрезается.
Кривые слои кренят конструкцию; когда центр тяжести выходит за опору, башня
рушится. Цель — построить башню как можно выше (игра «на рекорд»).

Ключевая идея проекта: **одно физическое ядро + сменные темы (скины)**. Темы
меняют только арт, анимацию и звук — правила и код общие. Это делает добавление
новых тем дешёвым и удерживает скоуп.

## Жанр и референсы

Балансовый стэкер на настоящей физике. Референсы по ощущению: Tower Bloxx,
Stack Tower. Это НЕ классический Stack с обрезкой свеса — здесь свес остаётся и
влияет на устойчивость.

## Ядро (общее для всех тем)

1. Сверху появляется объект-носитель (рука), держащий укладываемый предмет.
2. Носитель едет влево-вправо над башней.
3. Тап → предмет падает и ложится на башню как есть (без обрезки).
4. Работает настоящая 2D-физика твёрдых тел: вес, наклон, трение, столкновения.
   Неровная укладка накапливает крен.
5. Центр тяжести вышел за опору → башня падает → конец попытки.
6. Очки = высота башни (число успешно уложенных предметов / высота в условных
   единицах).

Решение по физике: используем **настоящую физику** (RigidBody2D в Godot), а не
аркадную имитацию. Принятый риск: тюнинг «честности» падений потребует времени.

### Разделение арта и физики

У каждого укладываемого предмета есть:
- тело физики с простой формой столкновения (прямоугольник/полигон);
- спрайт поверх тела (картинка темы).

Картинку можно менять, не трогая физику. Это технический фундамент системы тем.

## Система тем (скины)

Один движок + подключаемые пакеты ресурсов: арт + анимации + звук. Правила
идентичны во всех темах.

- Тема 1 (MVP): **Камни** — рука возит камень, тап → опускает на башню.
- Тема 2 (позже): **Блины** — рука с лопаткой возит блин, тап → шлёп на стопку.
- Дальнейшие темы добавляются как контент, без переписывания ядра.

Единый приём во всех темах: предмет несёт «рука-носитель».

## Платформы

Web-first. Один билд из Godot:
- экспорт в HTML5 → **Telegram Mini App**;
- тот же проект → нативный **Android** для Google Play (iOS — позже).

## Монетизация

Офлайн, без игровых серверов. (Для Telegram-версии нужен дешёвый хостинг под
веб-билд — это не игровой сервер, а просто раздача статики + бот.)

- Реклама за награду: «продолжить башню после падения за просмотр ролика».
- Межстраничная реклама между попытками (нечасто, чтобы не раздражать).
- Разовая покупка «убрать рекламу».
- Без подписок (чтобы не плодить обязательства и серверную часть).

## Движок и инструменты

- **Godot 4** — встроенная 2D-физика, экспорт в веб и в нативные сторы из одного
  проекта. Портативный, ставить отдельно ничего не нужно.
- Позже: Web-шаблоны экспорта (из Godot), Android SDK + ключ подписи для Google
  Play.
- Git — уже есть.

## Объём MVP (первый прототип)

В скоупе:
- физическое ядро (носитель ездит, тап роняет, башня кренится и падает);
- одна тема — «Камни» (на заглушках-формах от ассистента);
- счётчик высоты;
- экран «ещё раз» после падения.

Вне скоупа MVP (потом):
- меню, магазин, «убрать рекламу»;
- вторая тема (блины);
- реальный продакшн-арт и звук;
- интеграция рекламы;
- экспорт в Telegram и Google Play.

Критерий успеха MVP: «весело падать» — играется и хочется ещё раз, до всякого
арта.

## Арт-план

Ассистент делает только заглушки (простые формы, базовые спрайты, программные
эффекты вроде пыли при падении). Продакшн-арт берётся отдельно: ассет-сторы
(kenney.nl, itch.io), ИИ-генераторы или фрилансер.

Объекты темы «Камни» для MVP:
- рука-носитель;
- камень (2–3 вида);
- основание/постамент;
- фон;
- эффект пыли при падении;
- UI: счёт, кнопка «ещё раз», переключатель звука.
